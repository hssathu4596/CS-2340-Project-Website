GreenPlate: A Sustainable Food Management System

Welcome to GreenPlate, your ultimate solution for sustainable eating and shopping habits!

Introduction

GreenPlate is more than just an app; it's a movement towards a more sustainable lifestyle. In a world where food waste and inefficient resource utilization are pressing concerns, GreenPlate emerges as a beacon of change. Our app is dedicated to fostering sustainable eating and shopping habits by empowering users to make conscious choices that benefit both the planet and their well-being. GreenPlate boasts a user-friendly interface designed to streamline your sustainable food management journey. Simply create a secure account, log your meals, explore our extensive recipe database, and let GreenPlate do the rest. With intuitive features and actionable insights, our app empowers you to make informed decisions that benefit both your health and the environment.

Key Features

  • Food Tracking: Track your daily calorie intake and monitor your nutritional goals effortlessly.
  • Recipe Management: Create, save, and cook delicious recipes while managing ingredient quantities efficiently.
  • Pantry Organization: Keep tabs on your ingredient inventory and minimize food waste by utilizing what you already have.
  • Shopping List Generation: Automatically generate shopping lists based on missing recipe ingredients, ensuring you only buy what you need.
  • Actionable Insights: Receive personalized recommendations and insights related to recipes, ingredients, and purchasing needs, making meal planning a breeze.

Design and Architecture

GreenPlate's architecture is designed around the Model-View-ViewModel (MVVM) architecture pattern, leveraging Android Studio and Java to create a robust and maintainable application. MVVM separates the user interface logic from the business logic, promoting a clean and organized codebase that is easy to understand, test, and maintain.

Components

  • Model:
    • Represents the data and business logic of the application.
    • Encapsulates data sources, such as our Firebase database.
    • Provides methods for retrieving, updating, and deleting data.
    • Acts as classes for ingredients, meals, and other entities.
  • View:
    • Represents the user interface components of the application.
    • Observes changes in the ViewModel and updates the UI accordingly.
    • Delegates user interactions to the ViewModel for processing.
  • ViewModel:
    • Acts as an intermediary between the Model and the View.
    • Contains the presentation logic and state of the UI.
    • Retrieves data from the Model, processes it, and exposes it to the View via observable properties.

Design Patterns

  • Observer/Observable Pattern:

    We utilize the Observer design pattern through the utilization of MutableLiveData objects that provide the connections between classes. For example, the cookbook monitors the pantry and updates every time there are ingredients added or removed from the pantry. The cookbook calculates whether there are sufficient ingredients and indicates to the user whether they would be able to cook the recipe. In this case, the pantry serves as the observable part of the pattern and the cookbook servers as the observer.

  • Singleton Pattern:

    We made the User class a singleton because it makes sense to have only 1 user at a time logged into the app. Each time a user logs into the app, a new User object is created for the user that is logged into the app. There should not be multiple users logged into the app on the same device at a time, so making the User class a singleton prevents this from happening by only allowing 1 user object to be created.

  • Strategy Pattern:

    This code snapshot demonstrates the use of the Strategy design pattern because the RecipeFragment class can dynamically choose from the filtering strategies at run time depending on the way the user wishes to sort their recipes. The three concrete strategies are the different ways that the user can choose to filter their recipes and are depicted by the classes CalorieFilter, IngredientCountFilter, and NoFilter which all implement the FilterStrategy interface. The context part of the Strategy Design Pattern is demonstrated through the use of the FilterContext class which contains a reference to the filter strategy being used currently. By using the Strategy Design Pattern in our code, we give our users the ability to select different ways to view their recipes and ensure that our code is easily maintainable and extendable if we would like to add more ways to sort the recipes in the future.

By adopting the MVVM architecture pattern and implementing various design patterns within Android Studio using Java, GreenPlate achieves a well-structured, maintainable, and scalable codebase. The separation of concerns between the Model, View, and ViewModel enables developers to work efficiently and collaboratively, while also ensuring a smooth user experience for GreenPlate's users. With a focus on clean architecture and best practices, GreenPlate is poised to deliver a sustainable food management solution that makes a positive impact on both users and the environment.
